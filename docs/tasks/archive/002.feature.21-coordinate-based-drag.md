# 002.feature.21: Coordinate-Based Drag

**Status:** Pending
**Priority:** High
**Complexity:** Low

## Objective

Add a `dragCoordinates` command that performs a human-like drag operation using pixel coordinates instead of CSS selectors. This enables interaction with elements that cannot be targeted by selectors, such as captcha sliders, canvas-based UIs, and elements inside cross-origin iframes.

## Context

The existing `drag` command requires CSS selectors for both source and target elements. This fails when:

- **Captcha sliders** render custom UI that the DOM analyzer cannot detect as interactive elements (no `button`, `input`, `[role]`, etc.)
- **Canvas elements** where drag targets exist only as rendered pixels, not DOM nodes
- **Cross-origin iframe content** where selectors cannot reach into the frame
- **Pages with 0 detectable elements** (e.g. bot verification screens that strip the page)

The Abra user-testing platform hits this when Etsy (and similar sites) present a "slide to verify" captcha. The LLM can visually identify the slider handle and target in a screenshot, but needs coordinate-based drag to interact with it.

### Why `evaluate()` Doesn't Work

JavaScript-dispatched pointer/mouse events have `Event.isTrusted = false`. Captcha systems specifically check for trusted events. Only Playwright's CDP-level `page.mouse` API produces trusted events that satisfy this check.

## Technical Details

### Implementation

The cursor already supports coordinate-based movement via `moveToCoords(x, y)` which uses `page.mouse.move()` internally. The drag command uses `page.mouse.down()` and `page.mouse.up()`. We just need to combine these for coordinate-based drag.

### Changes Required

**`src/types.ts`** ‚Äî Add `'dragCoordinates'` to the `CommandAction` union type.

**`src/session.ts`** ‚Äî Add a new case to the command switch:

```typescript
case 'dragCoordinates': {
  const sx = params.sourceX as number;
  const sy = params.sourceY as number;
  const tx = params.targetX as number;
  const ty = params.targetY as number;

  // Move to source with human-like motion
  await cursor.moveToCoords(sx, sy);
  await new Promise(r => setTimeout(r, 400 + Math.random() * 200));

  // Press mouse down (trusted CDP event)
  await page.mouse.down();
  await new Promise(r => setTimeout(r, 100 + Math.random() * 100));

  // Drag to target with human-like motion
  await cursor.moveToCoords(tx, ty);
  await new Promise(r => setTimeout(r, 200 + Math.random() * 200));

  // Release mouse (trusted CDP event)
  await page.mouse.up();
  break;
}
```

**`src/fluent.ts`** ‚Äî Add a public method to the `Browser` class:

```typescript
async dragCoordinates(sourceX: number, sourceY: number, targetX: number, targetY: number): Promise<void> {
  await this.send('dragCoordinates', { sourceX, sourceY, targetX, targetY });
}
```

### How It Works

1. `cursor.moveToCoords(sx, sy)` ‚Äî Bezier curve movement to source, producing human-like motion with `page.mouse.move()` calls (trusted)
2. `page.mouse.down()` ‚Äî Trusted pointerdown/mousedown at source coordinates
3. `cursor.moveToCoords(tx, ty)` ‚Äî Smooth drag motion to target (mouse remains held)
4. `page.mouse.up()` ‚Äî Trusted pointerup/mouseup at target coordinates

All events are trusted CDP-level input events that satisfy `Event.isTrusted === true`.

## Usage

### Fluent API

```typescript
const browser = await puppet({ headless: false });
await browser.goto('https://example.com/captcha');
await browser.dragCoordinates(300, 450, 700, 450); // Slide captcha handle right
```

### Session/Interactive Mode

```javascript
await sendCommand({
  action: 'dragCoordinates',
  params: { sourceX: 300, sourceY: 450, targetX: 700, targetY: 450 },
});
```

### From Abra (LLM-driven)

The LLM sees a captcha in a screenshot, estimates coordinates, and outputs:

```json
{
  "type": "drag",
  "sourceX": 583,
  "sourceY": 372,
  "targetX": 800,
  "targetY": 372
}
```

## Testing

- Test with a local HTML page containing a slider that logs `Event.isTrusted` on pointer events
- Verify that all dispatched events have `isTrusted: true`
- Verify human-like cursor motion between source and target coordinates
- Test edge cases: same source/target, very short drag, very long drag
