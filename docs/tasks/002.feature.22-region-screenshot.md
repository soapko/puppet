# Task: Region and Element Screenshot Capture

## Overview

Add the ability to capture screenshots of specific regions (clip area) or individual elements, rather than only full-page or viewport screenshots. Playwright supports both natively ‚Äî this is a thin wrapper exposing them through Puppet's session protocol and fluent API.

## Current State

`screenshot` command accepts `{ path?, fullPage? }` and returns a base64-encoded PNG of the full viewport (or full page if `fullPage: true`). There's no way to capture a specific region or a single element.

## Implementation

### 1. Extend Screenshot Options

**File:** `src/types.ts`

Add `clip` and `selector` to the screenshot action params:

```typescript
// clip: { x, y, width, height } ‚Äî capture a rectangular region of the viewport
// selector: string ‚Äî capture a specific element (Playwright's element.screenshot())
```

### 2. Update Session Handler

**File:** `src/session.ts` (case `'screenshot'`)

```typescript
case 'screenshot': {
  let buffer: Buffer;
  if (params.selector) {
    // Element screenshot
    const element = await currentFrame.locator(params.selector as string).first();
    buffer = await element.screenshot();
  } else {
    // Viewport/region screenshot
    buffer = await page.screenshot({
      fullPage: params.fullPage as boolean,
      clip: params.clip as { x: number; y: number; width: number; height: number } | undefined,
    });
  }
  result = buffer.toString('base64');
  break;
}
```

Note: `clip` and `fullPage` are mutually exclusive in Playwright. If both are provided, `clip` takes precedence (drop `fullPage`).

### 3. Update Fluent API

**File:** `src/fluent.ts`

Extend the `screenshot()` method signature:

```typescript
async screenshot(options?: string | {
  path?: string;
  fullPage?: boolean;
  clip?: { x: number; y: number; width: number; height: number };
  selector?: string;
}): Promise<string>
```

Pass `clip` and `selector` through to the session command params.

### 4. Update REPL/CLI

**File:** `src/repl.ts` / `src/cli.ts`

Add `--clip x,y,w,h` and `--selector` flags to the `screenshot` command if applicable.

## API Examples

```typescript
// Full viewport (existing)
const png = await browser.screenshot();

// Region clip
const region = await browser.screenshot({
  clip: { x: 100, y: 200, width: 400, height: 300 },
});

// Element screenshot
const element = await browser.screenshot({ selector: '[data-testid="product-card"]' });

// Save to file
await browser.screenshot({ selector: '#chart', path: './chart.png' });
```

## Testing

- Viewport screenshot still works (regression)
- Clip region returns smaller image with correct dimensions
- Element screenshot captures just that element
- `clip` + `fullPage` together: `clip` wins, no error
- Invalid selector returns clear error
- Clip coordinates outside viewport: Playwright handles gracefully

## Notes

- Playwright's `page.screenshot({ clip })` is viewport-relative, not page-relative
- `elementHandle.screenshot()` auto-scrolls to the element if needed
- Both return PNG buffers by default ‚Äî no format changes needed
