# CDP Video Recording via Screencast

## Summary

Enable video recording for CDP-connected sessions (including Janus web tabs) using the CDP `Page.startScreencast` protocol. This bypasses Playwright's `recordVideo` limitation (requires context creation) by capturing frames directly via CDP and piping them to ffmpeg.

## Problem

Playwright's built-in video recording only works when the `BrowserContext` is created with `recordVideo` options. When connecting via `connectOverCDP()`, we attach to a pre-existing context — there's no API to retroactively enable recording. Currently, `videoEnabled` is hardcoded to `false` for all CDP connections.

## Solution

Use the CDP `Page.startScreencast` → `Page.screencastFrame` → ffmpeg pipeline. This is exactly what Puppeteer uses internally for its `page.screencast()` API. Since we already use CDP sessions for the visual cursor and screenshots, extending this to screencast is natural.

### Architecture

```
Chrome (CDP)                    puppet                         disk
┌─────────────┐   screencastFrame   ┌──────────────────┐   stdin    ┌────────┐   file
│ Page.start  │──────────────────→  │ CDPScreenRecorder │─────────→ │ ffmpeg │──────→ video.webm
│ Screencast  │   (base64 JPEG)     │ - ack frames      │  (JPEG)   │ VP9    │
│             │ ←─────────────────  │ - duplicate frames │          │        │
│             │  screencastFrameAck │ - pipe to ffmpeg   │          └────────┘
└─────────────┘                     └──────────────────────┘
```

## Implementation

### Phase 1: CDPScreenRecorder module (`src/cdp-screencast.ts`)

New module that encapsulates the screencast-to-video pipeline.

```typescript
import type { Page } from 'playwright';
import type { CDPSession } from 'playwright';
import { spawn, type ChildProcess } from 'node:child_process';
import { mkdir } from 'node:fs/promises';
import { dirname } from 'node:path';

interface ScreencastOptions {
  /** Output video file path */
  outputPath: string;
  /** Frame rate for output video. Default: 25 */
  fps?: number;
  /** JPEG quality for screencast frames (0-100). Default: 80 */
  quality?: number;
  /** Max frame width (Chrome downscales). Defaults to viewport width */
  maxWidth?: number;
  /** Max frame height (Chrome downscales). Defaults to viewport height */
  maxHeight?: number;
}

class CDPScreenRecorder {
  private cdpSession: CDPSession;
  private ffmpeg: ChildProcess | null = null;
  private lastFrame: { buffer: Buffer; timestamp: number } | null = null;
  private fps: number;
  private started = false;
  private frameCount = 0;

  constructor(
    cdpSession: CDPSession,
    private options: ScreencastOptions
  ) {
    this.fps = options.fps ?? 25;
    this.cdpSession = cdpSession;
  }

  async start(): Promise<void> {
    // 1. Ensure output directory exists
    await mkdir(dirname(this.options.outputPath), { recursive: true });

    // 2. Spawn ffmpeg
    this.ffmpeg = spawn(
      'ffmpeg',
      [
        '-loglevel',
        'error',
        '-f',
        'image2pipe',
        '-vcodec',
        'mjpeg',
        '-framerate',
        String(this.fps),
        '-i',
        'pipe:0',
        '-vcodec',
        'vp9',
        '-crf',
        '30',
        '-deadline',
        'realtime',
        '-b:v',
        '0',
        '-an',
        '-threads',
        '1',
        '-y',
        this.options.outputPath,
      ],
      { stdio: ['pipe', 'pipe', 'pipe'] }
    );

    // 3. Listen for screencast frames
    this.cdpSession.on('Page.screencastFrame', this.onFrame);

    // 4. Start screencast
    await this.cdpSession.send('Page.startScreencast', {
      format: 'jpeg',
      quality: this.options.quality ?? 80,
      maxWidth: this.options.maxWidth,
      maxHeight: this.options.maxHeight,
      everyNthFrame: 1,
    });

    this.started = true;
  }

  private onFrame = async (event: {
    data: string;
    metadata: { timestamp: number };
    sessionId: number;
  }) => {
    // Ack immediately (flow control)
    void this.cdpSession.send('Page.screencastFrameAck', {
      sessionId: event.sessionId,
    });

    const buffer = Buffer.from(event.data, 'base64');
    const timestamp = event.metadata.timestamp;

    // Duplicate previous frame to fill time gap (constant FPS for ffmpeg)
    if (this.lastFrame && this.ffmpeg?.stdin?.writable) {
      const duration = timestamp - this.lastFrame.timestamp;
      const copies = Math.round(this.fps * Math.max(duration, 0));
      for (let i = 0; i < copies; i++) {
        this.ffmpeg.stdin.write(this.lastFrame.buffer);
        this.frameCount++;
      }
    }

    this.lastFrame = { buffer, timestamp };
  };

  async stop(): Promise<string> {
    if (!this.started) return this.options.outputPath;

    // 1. Stop screencast
    try {
      await this.cdpSession.send('Page.stopScreencast');
    } catch {
      // Session may already be closed
    }

    // 2. Remove listener
    this.cdpSession.off('Page.screencastFrame', this.onFrame);

    // 3. Write final frame(s)
    if (this.lastFrame && this.ffmpeg?.stdin?.writable) {
      // Write at least one copy of the last frame
      const copies = Math.max(1, Math.round(this.fps * 0.5)); // half-second tail
      for (let i = 0; i < copies; i++) {
        this.ffmpeg.stdin.write(this.lastFrame.buffer);
      }
    }

    // 4. Close ffmpeg stdin and wait for exit
    if (this.ffmpeg) {
      const exitPromise = new Promise<void>(resolve => {
        this.ffmpeg!.once('close', () => resolve());
      });
      this.ffmpeg.stdin?.end();
      await exitPromise;
      this.ffmpeg = null;
    }

    // 5. Detach CDP session
    try {
      await this.cdpSession.detach();
    } catch {
      // May already be detached
    }

    this.started = false;
    return this.options.outputPath;
  }
}

export { CDPScreenRecorder, type ScreencastOptions };
```

### Phase 2: Integrate into `connectCDP()` (`src/browser.ts`)

When `video` option is set on a CDP connection, create a `CDPScreenRecorder`:

```typescript
// In connectCDP():
const isCDP = true;
let screenRecorder: CDPScreenRecorder | undefined;

if (options.video) {
  const videoOptions = parseVideoOptions(options.video, viewport);
  if (videoOptions) {
    const cdpSession = await page.context().newCDPSession(page);
    const outputPath = `${videoOptions.dir}/${Date.now()}.webm`;
    screenRecorder = new CDPScreenRecorder(cdpSession, {
      outputPath,
      fps: 25,
      quality: 80,
      maxWidth: viewport.width,
      maxHeight: viewport.height,
    });
    await screenRecorder.start();
  }
}

return {
  browser,
  context,
  page,
  videoEnabled: !!screenRecorder,
  cleanup: ...,
  screenRecorder,  // Add to BrowserInstance
};
```

## Dependencies

- **ffmpeg**: Required on the system PATH. Not bundled — users must install it.
  - macOS: `brew install ffmpeg`
  - Linux: `apt install ffmpeg` / `dnf install ffmpeg`
  - Check at startup and provide clear error if missing

## Output Format

- **WebM** with VP9 codec (matches Playwright's default output)
- CRF 30, realtime deadline (fast encoding)
- 25 FPS default

## Edge Cases

1. **ffmpeg not installed**: Check before starting, throw clear error
2. **Page navigation during recording**: Screencast survives navigations (Chrome-level)
3. **Tab switching**: Recorder is per-page — if user switches tabs, recording follows the original page
4. **CDP session disconnect**: `stop()` handles gracefully with try/catch
5. **Short recordings**: Ensure at least a few frames are written so ffmpeg produces valid output
6. **CDP proxy connections**: Screencast works through the CDP proxy since it goes through Playwright's CDP session API, not raw WebSocket
